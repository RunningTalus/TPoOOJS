Object Patterns

Mixins

Pseudoinheritance with Mixins

Mixins - occur when one object acquires properties of another WITHOUT modifying the prototype chain.

The first object a (receiver) receives the properties of the second object the (supplier) by copying those properties directly.

Mixins are shallow copies, so if a property contains an object, then both the supllier and the receiver will be pointing to the same object.

This mixin pattern is used for adding new behaviors to JS objects that already exist on other objects.
